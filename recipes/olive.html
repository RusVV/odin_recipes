Описание

При написании кода рекомендуется делать коммиты как можно раньше и чаще. Фиксируйте каждый раз, когда у вас есть значимое изменение в коде. Э то создаст временную шкалу вашего прогресса и покажет, что ваш готовый код не появился из ниоткуда.

Когда вы создаете свой проект, вам, вероятно, придется выполнить несколько циклов git add+ git commit, прежде чем вы будете готовы отправить его на GitHub с помощью git push origin main.
